🎥 MJPEG Stream Test — วัดปึก

ทดสอบการรับสัญญาณ MJPEG จาก MediaMTX ผ่าน HTTP — delay ต่ำกว่า HLS

วิธีใช้: ใส่ URL ของ MJPEG stream จาก MediaMTX เช่น
http://<windows-server-ip>:8889/cam1/stream
หรือถ้า MediaMTX เปิด MJPEG ไว้ที่ port 8888:
http://<windows-server-ip>:8888/cam1/stream.mjpeg

ถ้ายังไม่ได้เปิด MJPEG บน MediaMTX — ดูคำสั่ง FFmpeg ด้านล่าง
เพิ่มกล้อง:

🔧 ถ้า MediaMTX ยังไม่ได้เปิด MJPEG — รัน FFmpeg บน Windows Server

รัน 1 process ต่อ 1 กล้อง:

ffmpeg -i rtsp://100.93.6.53:8554/cam1 -f mjpeg -q:v 5 -r 15 -vf scale=854:480 pipe:1

หรือถ้าเปิด MJPEG ใน mediamtx.yml เพิ่ม:

paths:
  cam1:
    source: rtsp://100.93.6.53:8554/cam1

# เปิด HTTP server (ที่ port 8889 มี /stream endpoint อยู่แล้ว)
